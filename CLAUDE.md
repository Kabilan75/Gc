This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A Streamlit dashboard for UK gaming industry skill demand analysis. Four main tabs: UK Overview, AI Skill Gaps, Global Comparison, and CV Evaluator. Deployed on Streamlit Community Cloud from GitHub.

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Run locally
streamlit run app.py

# Regenerate cleaned global dataset (needed once after getting new Excel data)
python -m src.preprocess_combined_for_global
```

No build step, no test suite, no linter configured.

## Architecture

**Single-file app:** `app.py` (~3,100 lines) contains all UI, data loading, and chart logic. Supporting modules are small helpers:
- `city_to_country_tab5.py` — city→country mapping for the Global tab
- `preprocess_combined_for_global.py` — one-shot script to clean the raw Excel workbook

**Data pipeline (Step A → D):**
```
data/steps/step_a_clean_output.csv        ← raw skill inventory (1 row = 1 skill in 1 job ad)
data/steps/step_b_clustered_skills (2).csv ← skills + Cluster/Cluster_Name columns
data/steps/step_c_gap_scores.csv          ← Demand, Supply, Gap_Score per region per skill
data/steps/step_d_workshop_recommendations.csv ← ranked workshop recs per region
```
All four are loaded with `@st.cache_data` via `load_a/b/c/d()`. If any CSV is missing, the function falls back to hardcoded `_fallback_a/b/c/d()` DataFrames and labels the data as "demo sample" in the UI.

**Global tab data:** Prefers `Combined_Data_cleaned.xlsx` (git-ignored, generated by the preprocess script). Falls back to the raw `Updated_27_02_26_-_Kabilan.xlsx` (sheet "Combined Data").

**Tab routing:** Streamlit radio buttons in the sidebar select among the four tabs. Each tab is rendered in its own `if tab == "...":` block inside `app.py`.

## Key Patterns

**Color constants** (lines ~175–185): `TEAL = "#00E5CC"`, `PURPLE = "#A78BFA"`, `GREEN = "#34D399"`, `RED = "#FF5572"`, dark BG `#05090F`. All Plotly charts use `template="plotly_dark"`.

**`_dark(fig, h, margin_patch)`** — applies dark theme to any Plotly figure; **`show(fig, h, ...)`** — renders with `st.plotly_chart`.

**`_find(*names)`** — searches for data files in the project root, `data/steps/`, `data/`, and recursively. Use this when adding new data sources.

**`cn(skill)`** — normalizes skill display names (e.g. `"cpp"` → `"C++"`). Add entries to the `_NM` dict when new skills need display-name fixes.

**CV Evaluator** — lexical-only matching against Step A vocabulary. Skill aliases live in `CV_SKILL_ALIASES` (line ~348). Job role categories live in `CV_JOB_CATS` (line ~265). PDF extraction uses pdfplumber with PyPDF2 fallback.

## Gotchas

- Step B filename has a space and parentheses: `step_b_clustered_skills (2).csv`. `load_b()` handles this but don't rename the file without updating the loader. The file lives at `data/steps/step_b_clustered_skills (2).csv`.
- Wales and N. Ireland rows in Step C are sparse (<25 rows); the app shows a warning when these regions are selected.
- `Combined_Data_cleaned.xlsx` is in `.gitignore`. The global tab will silently fall back to the raw workbook if the cleaned file is absent.
- The `"game-texts"` skill is filtered out of Step A at load time — this is intentional.
